一、前言：案例背景與學習說明

1.1 工程概要

1.2 本手冊的學習目標

透過本案例的四項 EAGLE 系統任務，你將：

  1. 理解工程專案管理的完整脈絡：從專案建立、每日施工記錄、進度計算到品質查驗，體驗監造工程師的日常工作。

二、你的角色：監造工程師

在本系列任務中，你將扮演任泰技術顧問有限公司的監造工程師，負責監督「磺港溪再造步道整建工程」的施工品質、進度與安全。

2.1 監造工程師是誰？做什麼？

在公共工程中，「監造」是由主辦機關委託的工程顧問公司派駐現場的工程師，其角色介於業主（主辦機關）與承攬廠商（營造廠）之間。監造工程師的主要職責包括：

  1. 監督施工品質：確認承攬廠商是否依照設計圖說及規範施工，執行施工抽查、材料檢驗。
  2. 管控工程進度：記錄每日施工情形，核對預定與實際進度，發現落後時提出因應措施。
  3. 審核工程文件：審查承攬廠商送審的施工計畫、品質計畫、材料送審資料等。
  4. 維護工地安全：監督承攬廠商落實職業安全衛生管理，記錄並追蹤缺失改善。
  5. 編製監造報告：定期彙整施工情形，向主辦機關提報監造紀錄與建議。

三、什麼是「當責」？

「當責」（Accountability）與「負責」（Responsibility）不同。負責是完成被交付的工作；當責則是對最終結果負完全責任，即使過程中遇到困難，也要主動尋找解決方案，而不是等待他人指示。

在本系列任務中，「當責」的具體表現包括：

  1. 準確性：填寫的每一筆資料都必須經過確認，不能想當然耳地隨意填寫。
  2. 完整性：不遺漏任何必要的欄位或資訊，了解每個欄位存在的意義。
  3. 時效性：在規定時間內完成任務，理解工程現場「逾期」的嚴重性。
  4. 主動性：遇到不確定的問題時，主動查找資料、詢問 AI 助教或與同學討論，而非放棄或隨便填答。
  5. 反思性：完成任務後，回顧自己的過程，誠實面對不足之處，思考改進方法。
當責宣言 在開始任務之前，請你在心中對自己說：
「我是這個專案的監造工程師。我填寫的每一筆資料、做出的每一個判斷，都關係到這個工程的品質與安全。我將對我的工作成果負完全責任。」

四、EAGLE 系統簡介與登入

4.1 什麼是 EAGLE 系統？

EAGLE（Engineering Administration and General Ledger Expert）是一套雲端化的工程智慧管理系統，由計畫主持人與工程顧問公司合作開發。EAGLE 以監造及專案管理的視角設計，涵蓋施工日誌、自主檢查、缺失改善、送審管制、檢試驗管制、進度報告、估驗計價等功能模組。

4.2 系統功能架構

EAGLE 系統依使用情境分為三大模組：

模組主要功能
廠商施工執行施工日誌、自主檢查、缺失改善、檢試驗管制、送審管制、進度報告、估驗計價、檔案管理等
廠商成本分析詳細價目表上傳、價格庫管理
照片管理施工照片上傳、標籤管理、地圖檢索

4.3 登入與基本操作

  1. 開啟瀏覽器，輸入教師提供的 EAGLE 系統網址。
  2. 在登入頁面輸入教師發放的帳號與密碼。你將以「專案負責人」（監造工程師）身分登入。
  3. 登入後，系統會顯示你所屬的專案列表。點選你的磺港溪工程專案即可進入。
  4. 若需切換模組，請點選畫面左上角的藍色方塊，選擇「廠商施工執行」、「廠商成本分析」或「照片管理」。

1.5 任務執行步驟

步驟 A：新增專案

  1. 登入 EAGLE 系統，確認你是以「訂閱者」或「專案負責人」身分進入。
  2. 在系統首頁點選「新增專案」，選擇「一般式工程」。
  3. 依照開工報告表中的資料，填入以下欄位：
    • 工程名稱：磺港溪再造步道整建工程（奇岩抽水站至西安橋）
    • 契約編號：H-111-03-111166
    • 主辦機關：臺北市政府工務局水利工程處
    • 承攬廠商：永溢營造股份有限公司
    • 契約金額：22,700,000 元
    • 工期：日曆天 180 天
    • 開工日期：112 年 1 月 30 日
    • 預定竣工日期：112 年 7 月 28 日
  4. 確認所有欄位填寫正確後，儲存專案。新增的專案會顯示在清單最上方。

步驟 B：匯入契約工項

  1. 進入專案後，點選左側選單中的「契約工項」功能。
  2. 使用「匯入契約」功能，上傳原契約 CSV 檔案（磺港溪...原契約.csv）。
  3. 上傳後，系統會自動解析契約項目。請逐一確認以下項目是否正確匯入：
    • 一（一）整備工程：共 15 項工項
    • 一（二）土建景觀工程：包含混凝土、鋼筋、模板等主要結構工項
    • 確認各工項的單位、數量與單價是否與原始資料一致
  4. 核定契約：確認內容無誤後執行「核定」。注意：核定時間應設為決標時間。核定完成後僅能檢視及編輯，無法刪除或更改契約，須將整個專案刪除才能重來。請務必仔細確認後再核定。

步驟 C：設定成員與權限

  1. 進入「成員與權限」功能。
  2. 新增成員並設定角色（依教師指示新增小組成員帳號）。
  3. 確認各成員的權限等級：訂閱者（公司負責人）、專案負責人、一般使用者（文書人員）各有不同的操作權限。

步驟 D：匯入自主檢查類型

  1. 進入「自主檢查類型」功能。
  2. 可選擇「歷史專案匯入」或「系統預設匯入」：
    • 歷史專案匯入：從系統中既有的專案範本匯入檢查類型。
    • 系統預設匯入：匯入系統內建的標準檢查類型模板。
  3. 參照監造計畫中的施工抽查程序，確認本工程需要的檢查類型至少包含：測量放樣、鋼筋工程、模板工程、混凝土工程、植筋工程、地坪鋪面等。
  4. 根據自主檢查樣板資料夾中的檔案（13.自主檢查類型/自主檢查樣板），確認每種檢查類型的查驗項目是否完整。

步驟 E：上傳進度表

  1. 進入「進度表」功能，點選「新增進度表」。
  2. 選擇進入方式：可點選中間列表或左上角列表。
  3. 根據需求選擇模式，上傳工程網狀圖（16.進度表/工程網狀圖.pdf）與進度曲線表（16.進度表/進度曲線表.pdf）。
  4. 確認上傳的進度表能正確顯示各工項的排程與 S 曲線。

步驟 F：設定其他管理項目

  1. 檢試驗項目管理：進入「檢試驗項目管理」功能，新增本工程需要的檢試驗項目（如混凝土抗壓試驗、鋼筋拉力試驗等），並將項目連結至對應的契約工項。
  2. 人機管理：進入「人機管理」功能，新增本工程的施工機具與人力項目。系統預設有四項基本項目，可依實際需求新增。新增的項目會在施工日誌中顯示。
  3. 樣板管理：確認施工日誌、缺失改善、送審管制、檢試驗管制等表報的樣板格式是否正確。若需調整，可下載預設樣板修改後重新上傳。預設檔案類型：施工日誌為 Excel；缺失改善、送審管制、檢試驗管制為 Word。

1.6 提示問題

  1. 工程契約中的「詳細價目表」與「單價分析表」各扮演什麼角色？兩者之間是什麼關係？

1.7 自我檢核清單

  • 專案基本資料（工程名稱、編號、金額、工期、日期）與開工報告表完全一致。

  • 契約已核定，核定時間為決標時間。
  • 成員與權限已依教師指示設定完成。
  • 自主檢查類型已匯入，涵蓋本工程的主要施工項目。
  • 進度表（網狀圖與 S 曲線）已上傳且可正常顯示。
  • 檢試驗項目已新增並連結至對應工項。
  • 人機管理項目已設定。
  • 各報表樣板已確認或調整完成。

2.1 任務目標

2.2 學習標的

  1. 理解公共工程監造日報表（第一聯）的格式、各欄位意義及填寫要求。
  2. 學習從歷史施工資料中擷取關鍵訊息，判斷每日施工重點。

  6. 培養按時填報與自我檢核的工作習慣。

2.3 角色與責任

你身為監造工程師，填寫的是監造日報表第一聯。與承攬廠商填寫的施工日報（第二聯）不同，監造日報表記錄的是你「監督」施工的情形，而非你「執行」施工的情形。你的日報表需要反映：

  • 本日承攬廠商進行了哪些重要施工項目？
  • 你監督了哪些施工活動？是否依照設計圖說施工？
  • 本日有無進行材料抽查或品質查核？結果如何？
  • 預定進度與實際進度各為多少？是否有落差？
當責提醒 監造日報表是工程施工期間最重要的法律文件之一。在工程爭議、驗收糾紛甚至司法訴訟中，監造日報表常被引用作為關鍵證據。如果日報表記載不實（例如記載已施工但實際未施工），監造人員可能面臨偽造文書的法律責任。即使在本課程的模擬練習中，也請你以這樣的嚴謹態度對待每一份日報表。

3.1 任務目標

運用課堂所學的 CPM（要徑法）和 EVM（實獲值管理法），結合磺港溪工程的真實工程資料，進行工程進度的計算與分析，並在 EAGLE 系統中填寫進度報告。透過此任務，你將從「填寫日誌」的執行層面，提升到「分析進度」的管理層面。

3.2 學習標的

  1. 運用 CPM 方法，根據工程網狀圖計算要徑、最早/最晚開始與完成時間、浮時。
  2. 運用 EVM 方法，計算計畫值（PV）、實獲值（EV）與實際成本（AC），分析進度偏差（SV）與成本偏差（CV）。
  3. 根據計算結果，在 EAGLE 系統中撰寫進度報告，提出專業的進度分析與建議。
  4. 理解進度管控對工程專案成敗的關鍵影響。
  5. 培養以數據為基礎的工程管理決策能力。

3.3 角色與責任

身為監造工程師，進度管控是你最重要的職責之一。你需要定期向主辦機關提報工程進度報告，分析承攬廠商的施工進度是否符合預定計畫，如有落後需提出原因分析與因應建議。進度報告的品質直接影響主辦機關對工程的掌控能力。

當責提醒 進度落後是公共工程最常見的問題之一。如果監造工程師未能及時發現進度落後並提出警示，等到問題擴大後再處理，不僅增加趕工成本，更可能影響工程品質。因此，進度報告不只是「記錄現狀」，更是「預警管理」的重要工具。你的每一次進度分析，都可能影響這個工程最終能否如期完工。

3.5 任務執行步驟

步驟 A：CPM 分析

  1. 根據工程網狀圖，繪製（或整理出）本工程的前置關係圖（Activity-on-Node 或 Activity-on-Arrow）。
  2. 標示每項工作的工期，並執行前推（Forward Pass）與後推（Backward Pass）計算：
    • 最早開始時間（ES）與最早完成時間（EF）
    • 最晚開始時間（LS）與最晚完成時間（LF）
    • 總浮時（TF）= LS - ES 或 LF - EF
  3. 辨識要徑（Critical Path）：浮時為零的路徑。
  4. 列出要徑上的工作項目，標示哪些項目在工程進度 30% 時已經在執行或即將開始。

步驟 B：EVM 分析

  1. 以契約金額為基礎，建立 EVM 分析表。選定分析的時間點（建議以「工程進度 30%」作為分析時間點）。
  2. 計算各指標：
    指標公式意義
    PV（計畫值）截至分析時間點，預定應完成工作的預算金額我們原本計畫做多少？
    EV（實獲值）截至分析時間點，實際完成工作的預算金額我們實際做了多少？
    AC（實際成本）截至分析時間點，實際花費的金額我們花了多少錢？
    SV（進度偏差）EV - PV正值=超前，負值=落後
    CV（成本偏差）EV - AC正值=節省，負值=超支
    SPI（進度績效指標）EV ÷ PV>1 超前，<1 落後
    CPI（成本績效指標）EV ÷ AC>1 節省，<1 超支
  3. 根據估驗計價資料（9.估驗計價/磺港C-C(1).pdf 等），交叉比對 EV 與 AC 的計算結果。

步驟 C：在 EAGLE 系統中填寫進度報告

  1. 進入 EAGLE 系統的「進度報告」模組，點選「新增報告」。
  2. 在進度報告中填寫以下內容：
    • 報告期間與分析時間點
    • 預定進度 vs. 實際進度的比較
    • CPM 分析摘要：要徑項目、目前執行狀態
    • EVM 分析摘要：PV、EV、SV、SPI 等關鍵指標
    • 進度偏差原因分析（如有落後）
    • 後續建議與因應措施
  3. 下載進度報告確認格式與內容：點選「下載」，選擇報告匯出。
操作提示 進度報告的「新增」與「下載」操作與施工日誌類似。新增報告時，系統會提供結構化的欄位供你填寫。如有附件（如 EVM 計算表），可透過檔案管理功能上傳。

  3. 你在計算過程中犯過哪些錯誤？這些錯誤如果發生在真實工程中，可能有什麼影響？
  4. 「讓數據說話」對於工程管理有什麼重要性？與「憑經驗判斷」相比，各有什麼優劣？

3.9 任務產出

  2. EVM 分析表：包含 PV、EV、AC、SV、CV、SPI、CPI 的計算過程與結果。
  3. EAGLE 系統中的進度報告（含數據分析與建議）。
  4. 個人反思報告（200–300 字），包含：
    • CPM 與 EVM 在實務中如何幫助監造工程師？
    • 你在計算過程中遇到的困難與解決方法。
    • 對「以數據為基礎的工程管理」有什麼體會？

任務 4：自主查核表填寫 {% if current_user.is_authenticated %} 提交作業 {% endif %}

對應週次：第 9–10 週 預估時間：3–4 小時 難度：中等

4.1 任務目標

根據磺港溪工程的監造計畫與自主檢查樣板，在 EAGLE 系統中填寫施工品質自主查核表。透過此任務，你將理解公共工程三級品管制度中「監造查驗」的角色，學習如何執行施工品質的查核與記錄。

4.2 學習標的

  1. 理解公共工程三級品管制度的架構（承攬廠商自主檢查 → 監造單位查驗 → 主辦機關抽驗）。
  2. 認識本工程涉及的主要施工抽查程序（測量放樣、鋼筋、模板、混凝土等）。
  3. 學習自主查核表的格式與填寫方法，理解各查核項目的工程意義。
  4. 在 EAGLE 系統中操作新增查驗、填寫查核結果與下載查核表。
  5. 建立對施工品質「零容忍」的監造態度。

4.3 角色與責任

在三級品管制度中，監造工程師的角色是「第二級」品管——監造查驗。你的職責不是代替承攬廠商執行品質檢查，而是監督承攬廠商是否確實執行自主檢查，並針對關鍵施工項目進行抽查確認。

4.5 背景知識：三級品管與自主檢查

公共工程的三級品管制度是確保工程品質的核心機制：

層級執行單位內容
第一級承攬廠商（營造廠）施工自主檢查：承攬廠商在每個施工階段完成後，依據自主檢查表逐項檢核，確認品質符合規範後，報請監造查驗。
第二級監造單位（顧問公司）監造查驗：監造工程師接到承攬廠商的查驗請求後，到場進行抽查確認，填寫施工抽查紀錄表。
第三級主辦機關機關抽驗：主辦機關不定期到場抽驗，確認施工品質符合設計要求。

4.6 任務執行步驟

步驟 A：研讀自主檢查樣板

  1. 開啟自主檢查樣板資料夾（13.自主檢查類型/自主檢查樣板），瀏覽各類工程的檢查表。
  2. 重點研讀與磺港溪工程前 30% 進度相關的檢查表：
    • 自主檢查表——測量放樣
    • 自主檢查表——鋼筋
    • 自主檢查表——模板
    • 自主檢查表——混凝土
    • 自主檢查表——土方（整地）
  3. 理解每份檢查表中各查核項目的意義。例如，鋼筋查核表中「鋼筋號數、間距是否與圖說相符」這個項目，你需要理解為什麼鋼筋間距的正確性如此重要。

步驟 B：研讀歷史查核紀錄

  1. 開啟歷史自主檢查紀錄（3.自主檢查/C標抽查表.pdf），了解本工程實際執行過的查核內容。
  2. 注意觀察：
    • 查核頻率：多久執行一次查核？
    • 查核範圍：每次查核涵蓋哪些項目？
    • 查核結果：是否有發現不合格項目？如何處理？

步驟 C：在 EAGLE 系統中填寫自主查核表

  1. 進入 EAGLE 系統的「自主檢查」模組。
  2. 點選「新增查驗」，系統會要求選擇查驗的分項工程名稱（系統會根據契約工項自動帶入可選項目）。
  3. 選擇要查驗的施工項目後，系統會載入對應的自主檢查表格式。
  4. 依據歷史查核紀錄的內容，以監造工程師的視角填寫各查核項目：
    • 查核日期
    • 查核位置/區段
    • 各項目的查核結果（合格/不合格/不適用）
    • 監造意見與備註
  5. 完成後，使用下載功能匯出查核表：
    • 下載單筆查核表：選擇特定的查驗紀錄下載。
    • 下載查核總表：匯出所有查驗紀錄的彙整表。
操作提示
  • 在「新增查驗」時，分項工程名稱會根據你在任務 1 匯入的契約工項自動帶入，不需要手動輸入。
  • 下載查核表時，系統提供「單筆下載」和「總表下載」兩種方式，分別適用於個別查驗紀錄和彙整報告。

步驟 D：模擬缺失發現與改善（選做）

  1. 如果在查核過程中「發現缺失」（根據歷史缺失紀錄 4.缺失改善/缺失.pdf），可進入「缺失改善」模組進行練習。
  2. 首次新增缺失改善前，需先設定好編號前綴。
  3. 填寫缺失內容、缺失照片、改善期限，並追蹤改善狀態（待改善→已改善→已確認）。

4.7 提示問題

  1. 三級品管制度中，每一級各自的角色與責任是什麼？如果缺少其中一級，可能出現什麼問題？
  2. 「自主檢查」的「自主」是什麼意思？為什麼叫「自主」而不是「強制」？這背後有什麼管理哲學？
  3. 如果承攬廠商的自主檢查表上填的都是「合格」，但監造工程師到場抽查卻發現問題，這代表什麼？
